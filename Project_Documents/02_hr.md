# HR & User Management

## 1. Tổng quan

Module quản lý nhân sự, tổ chức, và tài khoản hệ thống.

**Người dùng liên quan:** Admin, Manager.

---

## 2. Business Rules

### 2.1 Cấu trúc tổ chức
```
Factory (Nhà máy)
 └── Department (Phòng ban: QC, Production, Engineering, Planning, Maintenance)
      └── Position (Chức vụ: Kỹ sư CNC, Inspector, Operator, Planner...)
           └── User (Nhân viên)
```

### 2.2 Trạng thái nhân viên (`work_statuses`)
- **Working** — đang làm việc, có thể gán vào ca/máy/NCR.
- **On Leave** — nghỉ phép, vẫn có tài khoản nhưng không hiện trong dropdown gán việc.
- **Resigned** — đã nghỉ, tài khoản bị khóa (`is_active = false`), dữ liệu lịch sử giữ nguyên.

### 2.3 User Types vs Roles
- **User Type** (`user_types`): xác định khả năng chuyên môn — "QC Inspector có thể nhập measure value và tạo NCR".
- **Role** (`roles`): xác định quyền truy cập menu trong PDM — "Engineer thấy menu Job, OP, Dimension nhưng không thấy HR".
- Một user có **1 user type** và **1 role**, hai thứ độc lập nhau.

### 2.4 Tạo tài khoản
- Admin tạo tài khoản → mật khẩu tạm được tạo tự động (random 8 ký tự).
- Mật khẩu tạm gửi qua email (nếu có) hoặc Admin thông báo trực tiếp.
- `first_login = true` → user bắt buộc đổi mật khẩu lần đăng nhập đầu.

### 2.5 Xóa / Vô hiệu hóa tài khoản
- **Không xóa cứng** tài khoản vì ảnh hưởng đến audit trail (ai tạo NCR, ai đo kiểm...).
- Khi nhân viên nghỉ: set `is_active = false` + `work_status = Resigned`.
- Tên vẫn hiện trong lịch sử với dấu hiệu nhận biết (ví dụ: "Nguyen Van A [Resigned]").

### 2.6 Phân công máy (Work Assignment)
- Một user có thể được phân công vào nhiều máy trong nhiều ca khác nhau (qua `shift_assignments`).
- Phân công này phục vụ cho: planning, tracking audit log tại máy nào.

## 3. Workflow

### Tạo nhân viên mới
```
Admin điền thông tin:
  - Họ tên, giới tính, email
  - Phòng ban → Chức vụ
  - User Type (QC, Operator, Engineer...)
  - PDM Role (quyền menu)
  - MES Role (quyền menu desktop)
  - Work Status = Working
→ Hệ thống tạo user_login (gợi ý từ tên) + mật khẩu tạm
→ Gửi email thông tin đăng nhập (nếu có SMTP)
→ User đăng nhập lần đầu → đổi mật khẩu
```

### Chuyển nhân viên sang phòng ban khác
```
Admin thay đổi department_id + position_id
→ Không cần thay đổi role (thường giữ nguyên)
→ Ghi confirm_log về thay đổi
```

## 4. Data Model

```sql
departments   (id, code, name)
positions     (id, code, description, is_active)
work_statuses (id, name, is_working)
user_types    (id, type_name, description, can_enter_value, can_raise_ncr)

users (
    id, user_login, password_hash, name, sex, email,
    user_type_id → user_types,
    position_id  → positions,
    work_status_id → work_statuses,
    role_id      → roles,
    mes_role_id  → mes_role_menus,
    first_login, reset_code, is_active,
    created_at, updated_at
)
```

## 6. Edge Cases

- **Xung đột user_login**: hai người cùng tên → thêm số phân biệt (nvana, nvanb).
- **Xóa department đang có user**: không cho xóa, phải chuyển user đi trước.
- **Thay đổi user_type**: phải xem xét ảnh hưởng — nếu user đang có measure values được nhập với type cũ, dữ liệu lịch sử giữ nguyên `user_type` snapshot (cột `user_type VARCHAR` trong `measure_values`).
- **Admin duy nhất**: không cho vô hiệu hóa tài khoản Admin cuối cùng còn hoạt động.

## UI Redesign — Phase H (đề xuất, chưa triển khai)

**`/hr` — layout 2 cột: org tree + user table**

- Cột trái: cây tổ chức **Department → Position**, mỗi node hiện số lượng user (`userCount`).
- Cột phải: bảng user hiện có — filter theo node đang chọn ở cây (`departmentId`/`positionId`, params đã hỗ trợ sẵn ở `GET /api/v1/users`).
- **API**: tận dụng `GET /api/v1/departments` + `GET /api/v1/positions` đã có. Cần thêm `userCount` per node — có 2 hướng:
  - (a) Thêm `userCount` vào `DepartmentDto`/`PositionDto` (additive field trong query handler, group-count từ `users`), hoặc
  - (b) Client-side: gọi `GET /api/v1/users?pageSize=9999` 1 lần rồi group-count tại frontend (đơn giản hơn, không cần đổi API — phù hợp quy mô 50-200 người).
- Click node "Department X" → bảng lọc tất cả user thuộc phòng ban X (mọi position); click node "Position Y" (con của X) → lọc thêm theo position.
